# CSE-ToolKit
So... this started as scattered Google Docs for myself, then became actual books. Don't ask how.

## What's Actually Here

| Thing | What It Is | Why I Made It |
|:---|:---|:---|
| **Cloud Computing** | AWS/Azure/GCP, Docker, K8s, serverless stuff | Got tired of paying for courses that teach less than the docs |
| **DevOps & Cloud** | CI/CD, Terraform, monitoring, the whole pipeline | Because "it works on my machine" isn't a deployment strategy |
| **DSA (Multi-Lang)** | Algorithms in C++, Java, Python, JS | Interview prep that doesn't lock you into one language |
| **DSA (C++ only)** | Same stuff, but deep in C++ | For when you actually need to care about memory |
| **OOP** | Classes, inheritance, patterns across languages | Saw too much bad object-oriented code |
| **System Design** | Scalable systems, real examples, interview BS | Senior interviews are a different game |
| **Interview Prep** | The full playbook coding, system design, behavioral | Because grinding LeetCode alone doesn't get you hired |

## The Real Story

I wrote these because I was annoyed. Annoyed at tutorials that skip the hard parts. Annoyed at interview books full of solutions you can't actually explain. Annoyed at paying $50 for a course when the instructor just reads documentation aloud.

So I started writing my own explanations. Then they got long. Then structured. Then *book-shaped*.

These are the guides I needed when I was learning. They're not perfect, but they're honest.
